Начинаем проект "Копия CalCom".

# Архитектура
1. Frontend - Vue3, Vite, TailWind, TypeScript, routes, Pinia
2. Backend - python, OpenApi, FastApi
3. База данных - MongoDB
4. Запуск через Docker, оркестрация docker-compose
5. Запросы с frontend - идут на frontend, и оттуда forward на бекенд

# Техническое:
- TypeSpec - как источник истины для контрактов между frontend и backend
- строгая типизация везде, где это разумно
- все ответы бекенда обернуты в Result-паттерн (ответ всегда поля type:success|failure и, соответственно, success либо failure)
- для тестов - фикстуры
- service - подход через UseCase (одна функция - один UseCase)

- бекенд, слои:
  - Controller - точки входа по REST
    - domain-данные с суффиксом Dto
  - Service - бизнес-логика
    - вход по UseCase
    - ответ UseCase - через Result-паттерн
    - domain-данные с суффиксом Obj
  - Repository - сохранение/чтение в базе данных
    - domain-данные с суффиксом Doc

- make - используем

# Стартовый функционал:
- фронтенд отображает статус бекенда - результат вызова /api/v1/health
- бекенд отдает /health
