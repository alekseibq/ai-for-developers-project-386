# Описание приложения
В проекте есть две условные роли: владелец календаря и гости.
При этом в проекте нет регистрации и авторизации.

## Роли

### Владелец календаря
Владелец календаря - один заранее заданный профиль.
Этот профиль по умолчанию используется в админской части.

### Гость
Гость бронирует слоты без создания аккаунта и без входа в систему.


## Функционал

### Владелец календаря

#### 1. Владелец календаря - создает типы событий.
Для каждого типа события задает id, название, описание и длительность в минутах.

#### 2. Владелец календаря - просматривает страницу предстоящих встреч
На этой странице в одном списке показаны бронирования всех типов событий.

### Гость

#### 1. Гость - создает бронирование
Это сценарий, который состоит из нескольких шагов
1. Гость просматривает страницу с видами брони
    - на этой странице, для каждого вида брони доступно/отображается: название, описание и длительность.
2. Гость выбирает тип события
3. Гость просматривает календарь
    - может выбрать день - тогда отобразится список доступных слотов (в этот день)
4. Гость выбирает свободный слот -> создает бронирование

### Ограничения
 - доступные слоты формируются на 14 дней, начиная с текущей даты
 - гость может записаться только на свободный слот из этого окна
 - на одно и то же время нельзя создать две записи, даже если это разные типы событий.

1. Obj и Dto - должны быть раздельные
