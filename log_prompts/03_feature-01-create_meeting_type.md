Сделаем фичу: "1. Владелец календаря - создает типы событий"
В базовой постановке было сказано:
- для каждого типа события Владелец календаря задает id, название, описание и длительность в минутах
- Владелец календаря - один заранее заданный профиль. Этот профиль по умолчанию используется в админской части

На бекенде у нас уже есть функционал для создания MeetingType.
Теперь нужно прикрутить UI на фронтенд.

1. В шапке добавляем ссылку "Админка"
   - ведет на страницу "Владельца календаря"
2. Добавляем раздел, похожий на "Личный кабинет владельца календаря"
   - route: /admin/*
   - Layout:
     - шапка остается как была
     - меню слева, пункты:
       - Типы событий
       - Предстоящие Брони
     - центр - соответствует тому, что выбрано в меню слева
     - по умолчанию, выбран 1-й пункт. То есть, "Типы событий"
3. Добавляем страницу "Типы событий"
   - внутри раздела "Личный кабинет владельца календаря"
   - route: /admin/meeting_types
   - отображает список "Типы событий" (MeetingType). Для каждого:
     - название
     - id
     - описание (более мелкий текст, чем для названия)
     - тайминг (красивенько, в виде tag с иконкой циферблата и кол-во минут)
   - кнопка "Создать новый Тип события"
     - ведет на /admin/meeting_types/new
4. Добавляем страницу "Создать тип события"
   - route: /admin/meeting_types/new
   - форма для полей MeetingType
   - кнопка "Отменить"
     - обратно на /admin/meeting_types
   - кнопка "Создать"
     - валидация на фронте
     - запрос на бекенд
     - с обработкой ошибок от бекенда
       - если криво заполнены поля
       - если бекенд недоступен
     - тостер с сообщением - как для success, так и для failure (но для failure и цвет поярче, и задержка подольше)
